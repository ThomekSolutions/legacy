Legacy Gameplay Systems

Legacy is a playable multiplayer browser RPG prototype. The current build focuses on a fast roguelike loop: create an heir, leave the haven, fight through shared depths, collect power, die permanently, and continue the dynasty.

Overview

Every player connects to an authoritative Node.js and WebSocket server. The server owns movement, attacks, enemy behavior, warps, loot collection, deaths, and persistence. The browser renders the Phaser client and sends input.

  • The game starts in Haven, a monster-free hub.
  • Combat happens in depth-based maps named combat, combat2, combat3, and so on.
  • The first combat map keeps the id combat for compatibility.
  • Depths currently scale up to Depth 100.
  • Connected players in the same world can see each other through server snapshots.

Character Creation

Before entering the game, the player chooses a hero name, a house name, and cosmetic appearance. The house represents the dynasty that persists after a character dies.

  • Names are cleaned by the server and capped to safe display text.
  • Every new character starts in Haven at level 1 with 100 HP, 0 XP, 0 gold, and 0 kills.
  • Starting power is 1 + floor(house renown / 35), so a successful dynasty gives future heirs a small head start.
  • Cosmetics are validated against the generated character catalog before being accepted by the server.
  • Available cosmetic slots include mount, cape, body, skin, hair, armor, hat, helmet, weapon, shield, pet, and aura.

Controls

Desktop

  • Move with WASD, ZQSD, or arrow keys.
  • Attack with left click or Space.
  • Attacks aim toward the mouse position.

Mobile

  • Use the left virtual joystick to move.
  • Use the right attack button or right-side touch area to attack.
  • The canvas disables browser gesture interference for play.

Worlds And Portals

Maps use 32-pixel tiles in an 80 by 80 world. Haven is the safe hub. Combat maps are procedurally carved paths and rooms using the current depth's biome.

  • Haven: no monsters, portal at tile 40,36 leads to Depth 1.
  • Combat entry: each combat map places the return portal at tile 40,72.
  • Forward travel: depths below 100 place an exit portal deeper in the map.
  • Return travel: Depth 1 returns to Haven; deeper maps return to the previous combat depth.
  • Exact tile rule: a player warps only when their current tile is a portal tile.
  • Portal cooldown: after warping, the player has a 1.2 second portal cooldown.
  • Healing return: returning to Haven restores up to 30 HP without exceeding max HP.

Biomes rotate through Ruined Outskirts, Marsh Causeway, Grave Barrens, Wildwood Hollow, Ashen Keep, and Cinder Vault. Depth affects enemy count, enemy strength, loot quantity, and rare loot odds.

Combat

Combat is real-time and server-authoritative. Players can swing in Haven, but attacks only damage enemies in combat worlds.

  • The server ticks at 60 Hz.
  • Base movement speed is 248 units per second.
  • Marsh terrain slows player movement to 176 units per second.
  • Player hitbox radius is 6.5.
  • Attack cooldown is 0.42 seconds.
  • The attack shape is a rectangle with range 39 and half-width 18.
  • Player damage is 19 + level * 5 + power * 4.
  • All enemies inside the attack rectangle can be hit by the same swing.

Enemies wander until a player is close. If a combat player is within 310 units, the enemy chases the nearest target. Enemy contact attacks use a 1.05 second cooldown and kill the player when HP reaches 0.

Enemy Scaling

  • Each combat depth starts with clamp(24 + floor(depth * 0.56), 24, 80) enemies.
  • Enemy level is based on depth, a random 1-4 level roll, optional level boost, and elite status.
  • Elite chance is clamp(0.04 + depth * 0.003, 0.04, 0.34).
  • Enemy HP is round((28 + level * 12 + depth * 8) * eliteMultiplier), where elite multiplier is 1.85.
  • Enemy damage is round((5 + level * 2.5 + depth * 0.7) * eliteMultiplier), where elite multiplier is 1.45.

Loot And Progression

Kills and loot are the main short-term progression systems. Power increases damage immediately, while level increases HP and damage.

  • Killing an enemy grants enemy level * 10 XP.
  • Killing an enemy grants 4 + enemy level gold immediately.
  • Level-up threshold is current level * 26 XP.
  • Leveling sets XP back to 0, increases level by 1, increases max HP by 16, fully heals the character, and adds 1 power.
  • Loot pickups are collected within 24 units.
  • Common loot grants 5 to 26 plus depth scaling gold and 1 power.
  • Rare loot grants 35 plus depth scaling and random bonus gold, and gives 2 or more power at very deep depths.
  • Enemy loot drop chance is clamp(0.25 + depth * 0.0015, 0.25, 0.4).
  • Rare loot chance is clamp(0.08 + depth * 0.0045, 0.08, 0.58).

Death And Legacy

Death is permanent for the current character. The player creates a new heir, but the account's dynasty continues through house records, renown, graves, and relics.

  • A grave records name, level, cause of death, survival time, kills, gold, and date.
  • The account keeps up to 30 personal graves.
  • The public grave list keeps up to 80 recent graves.
  • Renown gained on death is level * 3 + kills + floor(lifeSeconds / 30).
  • Account records track max level, longest life, and most kills.
  • If a character reaches level 3 or at least 8 kills, death can create a relic.
  • Relics are stored both on the account and in the public Hall of Legends list.

Technical Notes

  • The static client and WebSocket server run from server.js.
  • The Phaser game client is loaded only by play.html.
  • Server persistence is local JSON at data/legacy-state.json.
  • The browser keeps an account id in localStorage under legacy.accountId.v1.
  • The /api/legends endpoint exposes public records, recent graves, and relics.
  • The Hall of Legends displays server max level, most kills, longest life, oldest dynasty, recent graves, and relics.
